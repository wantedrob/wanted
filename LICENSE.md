## 🔧 Common Issues and Fixes

### ❌ "Auto rob loop starts but freezes mid-heist"
**Fix:** This typically occurs when a game update changes the heist trigger timing or UI sequence. Toggle **Auto Rob** off and back on to reset the sequence state. If it persists, download the latest version from the [Releases page](https://wantedrob.github.io/wanted/) — heist sequence handlers are updated with every game patch.

---

### ❌ "GUI doesn't appear after executing"
**Fix:** Ensure *Wanted* is fully loaded — your character must be active in the game world before executing. Press **Right Shift** (default GUI toggle) if the panel doesn't appear automatically. If that fails, rejoin the server and execute after a full clean load.

---

### ❌ "Infinite cash balance isn't increasing visibly"
**Fix:** Some *Wanted* server configurations have delayed HUD balance updates. The cash is being added — check your actual balance in the game's stats menu rather than the HUD display. Also verify *Auto Farm* and *Auto Rob* are both enabled, as the cash generation system requires active loops to function.

---

### ❌ "Anti-AFK isn't preventing the idle kick"
**Fix:** Reduce the anti-AFK interval in *Settings* → *Anti-AFK* → *Interval* to 30–45 seconds. Some servers use shorter custom idle detection timers. More frequent simulated input satisfies stricter server-side idle detection systems reliably.

---

### ❌ "Auto buy upgrades is purchasing items I don't want"
**Fix:** Open *Settings* → *Auto Buy* → *Skip List* and add the names of items you want to exclude from automatic purchasing. The auto buy system will skip those entries and move to the next eligible upgrade in the price order.

---

### ❌ "Speed boost causes my character to clip through walls"
**Fix:** Speed multipliers above 3x can cause physics clipping on certain *Wanted* map geometry. Reduce your multiplier to 1.5x–2x for stable traversal. If you need higher speed in open areas, manually increase it only when not near tight map structures.

---

### ❌ "Script stops working after my character respawns"
**Fix:** This should be handled automatically by the session persistence system as of v2.5.0. If you're on an older version, update to the latest release. If you're already on the latest and the issue persists, toggle all active features off and back on after respawn to manually re-initialize the loops.
